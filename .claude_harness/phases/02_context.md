# Phase 2: Context (Read-only)

## 목적
기존 코드베이스를 탐색하여 구현에 필요한 컨텍스트를 수집한다.

## 권한
- **Read-only**: 코드 읽기만 허용
- 파일 생성/수정 금지 (산출물 제외)

## 스킵 조건
- 프로젝트에 `lib/` 디렉토리가 없으면 (신규 프로젝트) **이 Phase를 스킵**한다
- 스킵 시 산출물에 "신규 프로젝트 — Context 스킵" 기록

## 수행 절차

### 1. 프로젝트 구조 파악
- `lib/` 하위 디렉토리 구조 확인
- Clean Architecture 레이어 (domain / data / presentation) 확인
- 기존 feature 목록 파악

### 2. 관련 코드 탐색
Phase 1의 요구사항을 기반으로:
- 수정/확장할 기존 파일 식별
- 재사용 가능한 위젯, 모델, 서비스 확인
- 의존성 (`pubspec.yaml`) 확인

### 3. 패턴 수집
- 기존 코드의 네이밍 패턴
- 상태 관리 방식 (Riverpod, BLoC 등)
- 라우팅 패턴
- 에러 처리 패턴

### 4. 영향 범위 분석
- 변경이 필요한 기존 파일 목록
- 새로 생성할 파일 예상 목록
- 잠재적 충돌/사이드이펙트
